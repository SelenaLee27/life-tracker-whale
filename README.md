# 轨迹 PWA

这是 `轨迹` 的 GitHub Pages 发布目录。上传本目录内的全部内容到仓库根目录即可。

## 1. 产品概览

`轨迹` 是一个单文件 HTML 本地生活记录应用，核心模块包括：

- 日程管理
- 日记
- 记账
- 总结
- 历史速览

应用以浏览器本地存储为主，支持 JSON 备份导出和导入。当前目标是保持单文件可用，同时为后续迁移到 PWA 或封装程序保留数据兼容基础。

当前应用为单页结构，底部四个主导航分别进入：

- 日程管理
- 日记
- 记账
- 总结

顶部提供：

- 应用标题 `轨迹`
- 彩色拖尾星辰装饰
- 历史速览入口
- 当前日期
- 随机鼓励话语
- 导出备份
- 导入备份

## 2. 全局能力

### 2.1 单文件运行

- 应用由一个 HTML 文件承载。
- CSS、JavaScript、界面结构都内嵌在同一个文件中。
- 不依赖网络。
- 可直接在浏览器打开使用。

### 2.2 本地保存

- 数据保存在浏览器 `localStorage`。
- 页面刷新后数据仍保留。
- 关闭浏览器后再次打开仍可读取。

### 2.3 数据备份

支持 `导出备份`：

- 导出为 JSON 文件。
- 文件名格式类似：`trajectory-backup-YYYY-MM-DD.json`。
- 导出内容包含应用标识、数据结构版本、导出时间、日程类别配置、记账类别配置、日程记录、日记记录、账单记录。

### 2.4 数据导入

支持 `导入备份`：

- 选择 JSON 文件导入。
- 导入前会提示确认。
- 导入后覆盖当前本地数据。
- 兼容旧版本数据结构。
- 支持将旧版固定分类数据迁移到新版动态分类结构。

## 3. Header 与全局交互

### 3.1 顶部布局

顶部包含：

- 左侧品牌区：`轨迹`、彩色拖尾星辰、历史速览入口。
- 右侧工具区：当前日期、随机状态文案、导出备份、导入备份。

### 3.2 随机状态文案

默认随机显示：

- `I can smile a little more`
- `要学会更加善待我自己`

执行操作后会短暂显示操作反馈，例如：

- 日程已提交
- 账单已删除
- 备份已导出
- 总结已更新

操作提示会在约 3 分钟后恢复为随机默认文案。

### 3.3 历史速览入口

- 顶部可进入历史速览。
- 历史速览不在底部主导航中。
- 历史速览作为独立视图显示。

## 4. 日程管理模块

### 4.1 动态日程类别

日程类别支持：

- 新增
- 改名
- 修改颜色
- 上移
- 下移
- 停用
- 删除并保留历史
- 迁移后删除

### 4.2 日程类别规则

当前规则：

- 最少启用 3 个日程类别。
- 最多保留 8 个日程类别。
- 类别名最多 6 个字。
- 不允许新增或改名为重复类别名。
- 每个类别必须有颜色。
- 类别排序会影响转盘显示顺序、日程卡片顺序、总结统计顺序、历史展示顺序。

### 4.3 默认日程类别

初始默认类别为：

- 本职工作
- 写作
- 爱护身体
- 投资
- 探索
- 偶像

默认类别保持存在，除非用户主动修改。

### 4.4 类别删除策略

日程类别支持两种删除方式。

`删除并保留历史`：

- 当前类别从可用类别列表删除。
- 历史日程记录保留原类别名。
- 后续不能再新建该类别日程。

`迁移后删除`：

- 用户选择迁移目标类别。
- 历史日程记录迁移到目标类别。
- 原类别删除。

### 4.5 日程输入规则

- 每天每个启用类别最多 1 条日程。
- 日程内容由用户手动输入。
- 提交前可自动保存草稿。
- 提交后正文锁定，不能继续编辑。

### 4.6 日程状态

每条日程支持三种状态：

- 未标记
- 已完成
- 未完成

界面上用 `✔`、`✘`、`○` 表示。

### 4.7 提交与删除

- 每条日程可单独提交。
- 提交后正文不可编辑。
- 状态仍可继续切换。
- 日程记录可删除。
- 删除前会二次确认。

### 4.8 转盘 UI

日程模块使用圆形转盘：

- 根据当前启用类别数量自动分区。
- 分区颜色来自类别颜色。
- 圆盘中部显示三行文案：`这一天`、`走过了`、`x个方向`。
- 每个分区显示类别名和状态：未提交、已提交、已完成。

### 4.9 折叠展开逻辑

- 默认不展开任何日程类别。
- 点击圆盘类别，只展开对应类别。
- 点击 `全部展开` 展示所有启用类别。
- 点击 `全部收起` 收起所有类别。
- 单个日程卡片可点击 `收起`。

### 4.10 转盘彩蛋

- 点击圆盘中心文字触发旋转动画。
- 转盘旋转约 10 秒后自动停止。

### 4.11 日程说明区

点击 `说明` 后：

- 显示日程规则。
- 显示分类设置。
- 隐藏转盘和日程卡片区域。
- 再次点击说明后恢复日程主界面。

## 5. 日记模块

### 5.1 每日一篇

- 每天最多一篇日记。
- 日记内容上限 200 字。
- 字数实时显示。

### 5.2 自动保存草稿

- 输入时自动写入本地数据。
- 未提交状态下可继续编辑。

### 5.3 提交规则

- 日记可提交。
- 提交后正文锁定。
- 提交后不可继续编辑。
- 日记可删除。
- 删除前会二次确认。

### 5.4 关联日程

日记模块支持 `关联日程`：

- 默认隐藏。
- 点击后展开。
- 根据当前日记日期显示同一天日程。
- 支持查看历史日期对应的日程。
- 即使日程类别被停用或删除，已有历史记录仍可显示类别名。

### 5.5 日记状态

日记会显示：

- 已提交 / 未提交
- 更新时间
- 当前字数

## 6. 记账模块

### 6.1 动态记账类别

记账类别支持：

- 新增
- 改名
- 修改颜色
- 上移
- 下移
- 停用
- 迁移后删除

### 6.2 记账类别规则

当前规则：

- 至少保留 1 个启用类别。
- 类别名最多 6 个字。
- 不允许新增或改名为重复类别名。
- 每个类别有独立颜色。
- 类别顺序会影响账单类别下拉框、汇总条形图、总结分析、历史记录排序。

### 6.3 默认记账类别

初始默认类别为：

- 交通
- 饮食
- 医药
- 演出
- 购物
- 公益

### 6.4 类别删除策略

记账类别删除必须迁移：

- 删除前选择目标类别。
- 该类别历史账单全部迁移到目标类别。
- 原类别删除。
- 不支持直接删除并保留旧类别。

### 6.5 新增账单

每笔支出包含：

- 日期
- 类别
- 金额
- 备注
- 唯一 ID
- 提交状态
- 创建时间
- 更新时间

### 6.6 唯一 ID

每笔账单都有唯一 ID：

- 后台保存。
- 界面不显示。
- 删除账单时按 ID 精准删除。
- 避免同一天同类别同金额的账单被误删。

### 6.7 默认显示当天支出

记账模块默认显示当前选择日期的支出。

### 6.8 日期区间筛选

支持：

- 开始日期
- 结束日期
- 应用筛选
- 恢复当天

筛选逻辑为闭区间，即包含开始日期和结束日期。

### 6.9 实时汇总

记账模块实时显示：

- 总支出
- 账单数量
- 各类别支出金额
- 各类别占比
- 彩色进度条

### 6.10 账单删除

- 单笔账单可删除。
- 删除前二次确认。
- 删除按钮较小，保留在账单卡片内。

## 7. 总结模块

### 7.1 日期区间总结

总结模块支持选择：

- 开始日期
- 结束日期

统计范围为闭区间，即包含开始日期和结束日期。

### 7.2 区间印象

区间印象会根据当前区间生成简短总结，参考：

- 高频日记关键词
- 日程提交与完成情况
- 主要支出类别

### 7.3 走得最多的方向

当前判定规则为：

- 优先比较提交次数。
- 提交次数相同再比较完成次数。
- 完成次数相同再比较最早完成时间。
- 仍相同则按类别名称排序。

### 7.4 日程完成情况

统计每个方向：

- 提交数
- 完成数
- 完成率
- 彩色进度条

展示格式类似：`2/3 · 67%`。

### 7.5 支出分析

统计区间内：

- 总支出
- 账单数量
- 各类别支出金额
- 各类别支出占比
- 彩色进度条

### 7.6 日记关键词分析

`也许在想` 模块会从日记内容中提取高频词：

- 支持中文词组。
- 支持英文词。
- 支持中英混合词，例如人名、昵称、代称。
- 采用长词优先。
- 会过滤被长词包含的短词，避免类似 `很开心 / 很开 / 开心` 重复出现。

### 7.7 关键词云

总结模块包含简单关键词云：

- 词频越高，字号越大。
- 使用彩色标签展示。
- 同时保留 Top 关键词列表。

## 8. 历史速览模块

### 8.1 按日期查看

历史速览支持选择日期，默认查看今天。

### 8.2 历史日程

- 类别名

- 日程内容

### 8.3 历史日记

显示所选日期已提交的日记：

- 日期
- 提交状态
- 日记正文

### 8.4 历史账单

显示所选日期已提交的账单：

- 类别
- 金额
- 备注

### 8.5 历史类别保留

如果某个类别后来被删除：

- 直接保留历史的类别名。
- 历史记录仍可查看。
- 不会因为类别设置变化而丢失文本语义。

## 11. 当前限制

### 11.1 存储限制

当前主存储仍是浏览器 `localStorage`：

- 清除浏览器站点数据可能导致记录丢失。
- 换浏览器或换设备不会自动同步。
- 长期使用建议定期导出 JSON 备份。

### 11.2 日记分析限制

关键词分析是前端轻量规则：

- 不是完整自然语言处理。
- 对复杂语义、同义词、情绪分析支持有限。
- 对人名和混合词已有增强，但仍可能漏检。

### 11.3 日程完成时间限制

新记录会保存 `firstCompletedAt`。旧记录如果没有该字段，会用已有更新时间或创建时间做兼容推断，因此旧数据的“最早完成”排序可能不是完全精确。
